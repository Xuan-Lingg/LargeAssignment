## 九、五大模块详细设计

### 9.1 模块一：RAG 知识库 (`modules/rag_engine.py`)

**职责**：将 PPT/实验指导书/高分报告向量化，在指导或评分时检索相关知识。

**核心类设计**：
```python
class RAGEngine:
    def build_index(docs_path, collection_name)    # 一次性构建向量索引
    def retrieve(query, collection_name, top_k=5)  # 语义检索
    def add_document(file_path, collection_name)   # 增量添加文档
```

**处理流程**：
1. `document_parser.py` 解析 pptx/pdf/docx/md → 统一为纯文本
2. 文本清洗：去重、去页眉页脚、合并短段落
3. 分块策略：512 tokens/块，128 tokens 重叠（防止语义断裂）
4. Embedding：使用本地 `bge-large-zh-v1.5`
5. 存入 ChromaDB，持久化到 `data/vector_db/`

**检索触发时机**：
- 报告拆解时：检索该实验对应的 PPT 知识点
- 指导生成时：检索相关实验的操作步骤和注意事项
- 评分时：检索评分标准和高分报告范例

---

### 9.2 模块二：工具调用 (`modules/tools.py`)

**职责**：为 Agent 提供可调用的外部工具。

| 工具名称 | 功能 | 实现方式 |
|---------|------|---------|
| `generate_flowchart` | 生成 Mermaid 流程图代码 | LLM 生成 Mermaid 语法 → 前端渲染 |
| `web_search` | 搜索在线知识补充 | DuckDuckGo Search API（免费） |
| `calculate_score` | 按评分维度计算总分 | Python 函数，加权求和 |
| `check_grammar` | 中文语法/表述检查 | LLM-based（prompt 驱动） |
| `export_markdown` | 导出分析结果为 Markdown | Python 字符串拼接 + 文件写入 |

**关键实现**：`generate_flowchart(mermaid_code: str) -> str`
- Agent 调用 LLM 生成 Mermaid 代码
- 返回的代码通过 Streamlit 的 `st.markdown()` 渲染
- Prompt 中约束 Mermaid 语法，避免生成非法图表

### 9.3 模块三：智能体执行 (`core/agent.py` + `modules/strategy_router.py`)

**职责**：接收用户意图，进行策略选择与校验，拆解任务并多步执行。

**双重判断机制**：

```
用户打开 UI
    │
    ├── [1] 从策略按钮中选择策略（UI 层面预设）
    │       可选策略：① 实验报告拆解指导
    │                 ② 完整报告评估打分
    │                 ③ 单个实验知识点讲解
    │                 ④ 自动检测（让 LLM 判断）
    │
    ▼
Agent 收到 (策略 + 报告内容)
    │
    ├── [2] LLM 语义二次校验
    │       判断策略是否与报告内容匹配
    │       ├── 匹配 → 确认策略，进入执行
    │       └── 不匹配 → 提示用户："检测到您选择的策略可能不适合，
    │                    建议使用 [推荐策略]，是否切换？"
    │
    ▼
确认策略后，Agent 进入任务执行流程
```

**策略-任务映射**：

| 策略 | Agent 执行流程 |
|------|---------------|
| **实验报告拆解指导** | 解析报告结构 → RAG检索相关PPT → 识别实验数量与类型 → 逐个实验生成：知识点/步骤/要点 → 可选生成流程图 → 汇总输出 |
| **完整报告评估打分** | 解析已完成报告 → RAG检索评分标准 → 逐维度评分 → 工具计算总分 → 对比高分报告 → 输出详细评语 |
| **单个实验知识点讲解** | RAG检索该实验的PPT → 深度展开知识点 → 生成思维导图 → 输出结构化讲解 |
| **自动检测** | LLM 先分析报告特征（空白多=待指导，完整=待评分），再路由到对应策略 |

**核心类设计**：
```python
class StrategyRouter:
    def select_strategy(user_choice, report_content)  # 双重校验
    def validate_strategy(strategy, report_content)    # LLM 判断策略是否匹配

class ReportAgent:
    def analyze_report(report_text)                    # 识别报告结构与实验
    def decompose_experiments(report_text)             # 拆解为实验列表
    def guide_experiment(exp_name, rag_context)        # 单个实验指导
    def evaluate_report(report_text, rag_context)      # 完整报告评估
```

### 9.4 模块四：多模态输出 (`modules/tools.py` 中 + `app.py` 渲染)

**职责**：生成可视化内容，包括实验流程图、知识思维导图、评分雷达图。

| 输出类型 | 技术方案 | 触发场景 |
|---------|---------|---------|
| 实验流程图 | Mermaid `graph TD` | 实验拆解时展示步骤关系 |
| 知识点思维导图 | Mermaid `mindmap` | 知识点讲解时展示层级关系 |

| 报告结构树 | Mermaid `graph` 或 JSON 树 | 报告结构分析 |

**前端渲染方式**：

```python
    <div class="mermaid">{mermaid_code}</div>

# Plotly 雷达图
import plotly.express as px
fig = px.line_polar(r=values, theta=dimensions, line_close=True)

```

### 9.5 模块五：模型微调 (`scripts/prepare_finetune_data.py` + 微调流程)

**职责**：基于已完成高分报告数据，微调小规模 LLM，使其内化评分标准和高分写作模式。

**方案 A：In-Context Learning（轻量，优先实现）**
- 不真正微调模型，而是将高分报告作为 Few-shot Examples 注入 Prompt
- 从 `data/reference_reports/metadata.json` 中选取最相关的高分范例
- 在评分/指导时作为参考上下文提供给 LLM

**方案 B：LoRA 微调（进阶，选做）**
- 使用 Qwen2.5-7B 或更小模型（如 Qwen2.5-1.5B）
- 构建训练数据：已完成报告文本 → 对应的评分维度分数 + 评语
- 使用 LLaMA-Factory 或 Unsloth 进行 LoRA 微调
- 微调后的模型能更准确地按照课程评分标准打分

**训练数据格式**：
```json
{
  "instruction": "请根据以下评分标准，对这份实验报告进行评分",
  "input": "报告内容...\n评分标准：1)目标与任务分解(10分) 2)架构设计(10分)...",
  "output": "{\"目标与任务分解\": 8, \"架构设计\": 9, ..., \"总评\": \"...\"}"
}
```

**实现步骤**：
1. 收集 ≥5 份已评分的高分报告
2. 人工或 LLM 辅助生成评分理由
3. 格式化为指令微调数据（30-50 条）
4. 使用 LLaMA-Factory 进行 1-3 epoch LoRA 训练
5. 对比微调前后评分一致性

### 流程一：实验报告拆解指导

```
学生上传待完成报告 ──► document_parser ──► 纯文本
                                              │
学生选择策略 "实验报告拆解指导" ────────────────┤

                              ▼

                              │
                              ├── RAG.retrieve(实验名称, "course_knowledge")

                              │
                              ├── RAG.retrieve(实验名称, "reference_reports")

                              │
                              ├── LLM.generate(知识点 + 思路步骤 + 要点)
                              │
                              └── LLM.generate_mermaid(实验流程图)
                              
                              ▼
```

## 十四、验证方法

1. **RAG 检索验证**：上传示例 PPT，用几个实验相关问题测试检索结果是否相关
2. **策略路由验证**：分别输入待完成报告和已完成报告，测试自动检测是否正确
