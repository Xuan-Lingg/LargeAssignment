# 实验五：智能体技能开发实验

## 一、实验目的

1. 理解大模型智能体的基本组成，包括大语言模型、工具/技能、提示词、任务规划、执行循环和日志记录。
2. 掌握使用 LangChain 定义工具并让智能体调用工具完成任务的方法。
3. 能够设计一个简单智能体系统，使其根据用户任务调用检索工具、计算工具、数据库接口或日志工具。
4. 能够记录并分析智能体的工具调用过程，判断工具选择是否合理、结果是否可靠。

## 三、实验内容

### 3.1 任务说明

本次实验要求你基于 LangChain 开发一个简单智能体系统。推荐主题为 **课程助教智能体**，也可以选择学习助手、实验报告助手、校园事务助手、资料问答助手等相近主题。

智能体需要能够接收自然语言任务，并根据任务内容自动选择合适工具完成任务。系统不能只写成固定流程函数调用，应体现“根据问题选择工具”的过程。

### 3.2 基本功能要求

你的智能体至少应包含以下功能：

1. **接入大模型 API**：使用自己的 API Key 调用大模型，代码中不得硬编码或泄露完整 API Key。
2. **技能/工具机制**：至少定义并注册 3 个工具/技能。
3. **任务执行流程**：实现“规划/判断 → 工具调用 → 观察结果 → 生成最终回答”的流程。
4. **执行记录**：保存或打印每一步工具调用过程，包括工具名称、输入、输出和最终回答。
5. **测试问题**：设计不少于 5 个测试问题，覆盖不同工具调用场景。

### 3.3 大模型 API 配置要求

建议在 `实验五环境/.env` 中配置以下环境变量：

```bash
LLM_API_KEY=你的API_KEY
LLM_BASE_URL=你的大模型厂商OpenAI-compatible接口地址
LLM_MODEL=你要调用的模型名
LLM_TEMPERATURE=0
```

注意：

- 不要把 `.env` 文件提交到作业平台。
- 不要在报告截图、代码注释或运行结果中暴露完整 API Key。
- 如果 API 调用失败，报告中应说明失败原因，并保证工具、规划和日志部分可以运行。

### 3.4 工具/技能要求

至少包含以下五类工具中的三种：

| 类型 | 说明 | 示例 |
|---|---|---|
| 检索/搜索工具 | 从本地资料、课程文档或网页摘要中检索信息 | 课程资料检索、政策文档检索、FAQ 搜索 |
| 计算工具 | 完成确定性计算 | 成绩计算、日期计算、公式计算、预算计算 |
| 数据库/表格查询工具 | 查询结构化数据 | 课程日程 CSV/SQLite 查询、实验任务表查询 |
| 日志保存工具 | 保存执行过程或用户提醒 | 保存学习计划、保存答疑记录 |
| 计划生成工具 | 根据目标生成任务计划 | 复习计划、实验完成计划 |

> 注意：如果平台网络或 API 不稳定，不要求一定实现真实互联网搜索，可以使用本地课程资料检索工具代替搜索工具。

### 3.5 数据与资料选择

可以选择以下方式之一：

**方式一：使用提供的课程资料**

直接使用 `docs/` 目录下的课程资料和 `data/course_schedule.csv`，开发课程助教智能体。

建议测试问题：

1. 课程总评成绩由哪些部分组成？
2. 平时成绩 90、实验成绩 86、期末成绩 78、课堂参与 95，总评是多少？
3. 第 13 周课程主题和任务是什么？
4. 实验迟交有什么规则？
5. 帮我生成一份第 13 周智能体实验的完成计划，并保存到日志。

**方式二：使用自己的资料**

准备自己的文档或结构化数据，要求：

- 文档数量不少于 2 篇，或结构化数据不少于 15 条记录；
- 资料内容应包含可检索、可计算或可查询的信息；
- 实验报告中说明资料来源和用途。

### 3.6 对比实验

完成基础智能体后，请至少选择两个维度进行对比：

1. **无工具 vs 有工具**：对比同一问题在不调用工具和调用工具时的回答差异。
2. **不同工具组合**：例如只有检索工具、检索+计算工具、检索+计算+数据库工具的效果差异。
3. **不同工具描述**：修改工具 description，观察模型是否更容易正确选择工具。
4. **不同测试问题类型**：比较事实查询、计算、日程查询、计划生成等任务的成功率。
5. **不同模型或参数**：在 API 额度允许的情况下，对比不同模型或不同 temperature 设置对回答稳定性的影响。

对比时需记录：用户问题、预期应调用工具、实际调用工具、工具输出、最终回答、是否正确。

### 3.7 结果分析

实验报告中回答以下问题：

1. 你的智能体包含哪些技能？每个技能适合解决什么问题？
2. 智能体在哪些问题上能正确选择工具？哪些问题上选择错误？
3. 工具调用结果是否明显改善了最终回答？请结合对比实验说明。
4. 系统有哪些不足？可以从工具描述、提示词、执行步数、错误处理、日志记录等角度分析。
